# HealthOS Territory documentation root

This folder stores repository-local documentation scaffolds for Territory records.

Territories are documented repository domains. They describe canonical docs, code or schema paths, invariants, known gaps, test expectations, validation commands, forbidden moves, maturity, and owner profile.

Territories are not clinical domains of authority. They do not move law out of HealthOS Core, do not make GOS sovereign, and do not make apps or engineering tools authoritative.

Canonical doctrine remains in:
- `docs/architecture/47-steward-settler-engineering-model.md`
- `docs/execution/19-settler-model-task-tracker.md`

## Intended layout

- `territories/` — future Territory record files.

Current maturity: documentation scaffold only.
