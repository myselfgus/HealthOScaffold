# CustomSDK

`CustomSDK` is the Tier 3 compliance SDK for governed Stage packages.

It is not a runtime, not Core law, and not a Stage. It gives each Stage a typed
manifest vocabulary for capabilities, consumed Boundary surfaces, degradation
behavior, validation requirements, and prohibitions before the Stage can be
accepted into HealthOS.

Current maturity: scaffold SDK. The source of truth is still each Stage
`Custom.md`; typed manifests and stricter validation can be added as the Stage
package split matures.
